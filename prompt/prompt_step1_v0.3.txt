Diátaxis示例如下：
根据 Diátaxis 文档结构方法，文档被划分为四种类型，每种类型的目标、读者和内容结构各不相同。下面我给出每种文档类型的简单中文示例，不使用特殊符号或表格，仅纯文本：

1. 教程（Tutorials）

目的：引导新手完成一次完整的体验。
示例：

你第一次使用这个博客系统？下面是创建你的第一篇博文的步骤：

第一步，打开终端，运行 blog startproject myblog 创建项目。
第二步，进入 myblog 文件夹，并运行 blog runserver 启动服务。
第三步，在浏览器中访问 http://localhost:8000，点击右上角的“新建博文”按钮。
第四步，填写标题和正文后，点击“发布”按钮。你就完成了第一篇博文的发布。

2. 指南（How-to guides）

目的：解决具体任务，操作导向。
示例：

如何添加自定义标签页：

首先，打开 config/settings.py 文件，找到 TABS 配置项。
然后，在末尾添加一个新的字典项，例如 {"name": "帮助", "url": "/help"}。
保存后，重新启动服务。浏览器中将显示新的“帮助”标签页。

3. 参考（Reference）

目的：提供准确、系统性的事实描述，偏向查阅手册。
示例：

配置项：AUTO_SAVE_INTERVAL

说明：控制自动保存的时间间隔，单位为秒。
默认值：300
取值范围：正整数
使用示例：设置为每10分钟自动保存：AUTO_SAVE_INTERVAL = 600

4.解释（Explanations）

目的：帮助读者理解系统背后的设计理念和原理。
示例：

为什么我们选择使用 Markdown 而不是富文本编辑器？

Markdown 保持了纯文本的简单性，让内容更易于版本控制、协作与长期维护。相比之下，富文本编辑器虽然操作直观，但产生的 HTML 内容复杂，难以在多人开发中统一格式。我们的目标是让博文的内容在不同平台上都能一致呈现，因此选用了 Markdown。
